🧠 KEY CONCEPTS
═════════════════════════════════════════════════════════════════════════════

Service Worker:  Intercepts requests, handles caching, enables offline
Manifest:        Tells browser how to display your app
Icons:           Show on home screen when installed
Cache:           Browser storage for offline access
Offline-First:   App works without network, syncs when connected

📊 CACHING STRATEGIES
═════════════════════════════════════════════════════════════════════════════

API Routes:      Network First → Try server, use cache if offline
Static Assets:   Cache First → Use cache, update in background
Pages (HTML):    Network First → Try server, use cached page if offline
Other:           Stale While Revalidate → Use cache, update silently

🔄 AUTO-UPDATES
═════════════════════════════════════════════════════════════════════════════

How it works:
1. App checks for new version every 60 seconds
2. If found, new Service Worker activates immediately
3. Next refresh loads new content automatically
4. Users don't need to manually update

To force update:
1. Edit: public/sw.js
2. Change: const CACHE_VERSION = 'v1' → 'v2'
3. Deploy
4. Old cache deletes automatically on next visit

🐛 COMMON ISSUES & FIXES
═════════════════════════════════════════════════════════════════════════════

Service Worker not showing?
  → Restart dev server: npm run dev
  → Hard refresh: Ctrl+Shift+R
  → Clear: DevTools → Application → Clear site data

Blank screen after install?
  → In console: navigator.serviceWorker.getRegistrations()
              .then(r => r.forEach(x => x.unregister()))
  → Restart app

Cache not updating?
  → Edit public/sw.js
  → Change CACHE_VERSION from 'v1' to 'v2'
  → Redeploy

Icons not showing?
  → Check files exist: public/icon-192.png, public/icon-512.png
  → Wait 1-2 minutes
  → Uninstall and reinstall app

App doesn't work offline?
  → Check DevTools → Service Workers (should be active)
  → Check DevTools → Cache (should have files cached)
  → See PWA_SETUP.md troubleshooting
